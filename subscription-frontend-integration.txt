2. COMPLETE API ENDPOINTS
================================================================================

  All endpoints are under the base path: /api/v1

  2a. PUBLIC (NO AUTH REQUIRED)
  ──────────────────────────────────────────────────────────────────────
  GET   /plans                                    List active plans
  POST  /clinics/register                         Register new clinic + owner
  POST  /payments/webhook                         Paymob webhook (server-side only)

  2b. CLINIC OWNER (WEB DASHBOARD)
  ──────────────────────────────────────────────────────────────────────
  POST  /auth/login-web                           Web login (returns 403 if no subscription)
  POST  /subscriptions/initiate-payment           Initiate Paymob subscription payment
  GET   /subscriptions/my                         View current subscription
  POST  /subscriptions/my/cancel                  Cancel my subscription

  2c. STAFF / DOCTOR (WEB DASHBOARD)
  ──────────────────────────────────────────────────────────────────────
  POST  /auth/login-web                           Returns 403 if clinic has no active subscription

  2d. SUPER ADMIN
  ──────────────────────────────────────────────────────────────────────
  GET   /admin/dashboard/clinics/pending          List clinics pending approval
  POST  /admin/dashboard/clinics/{id}/approve     Approve clinic registration
  POST  /admin/dashboard/clinics/{id}/reject      Reject clinic registration (with optional reason)
  GET   /admin/plans                              List ALL plans (incl. inactive)
  POST  /admin/plans                              Create a new plan
  PUT   /admin/plans/{id}                         Update a plan
  DELETE  /admin/plans/{id}                       Delete a plan (soft-delete)
  GET   /admin/dashboard/subscriptions            List all subscriptions (paginated + filters)
  POST  /admin/dashboard/subscriptions/{id}/revoke  Revoke an active subscription

3. USER FLOWS
================================================================================

  FLOW A: NEW CLINIC REGISTRATION (BRAND NEW USER)
  ──────────────────────────────────────────────────────────────────────

  Landing Page / Marketing Page
       │
       ▼
  [GET /api/v1/plans] — Show pricing cards
       │
       ▼
  User clicks "Get Started" or "Subscribe"
       │
       ▼
  [POST /api/v1/clinics/register]
  Body: { fullName, email, phoneNumber, password,
          clinicName, clinicAddress, specializationId, ... }
       │
       ▼
  Response: 202 Accepted — { userId, message, isPendingApproval: true }
       │
       ▼
  Show "Registration submitted for approval" screen
  ─── Wait for SuperAdmin to approve (no real-time notification yet) ───
       │
       ▼
  SuperAdmin approves via admin panel
       │
       ▼
  User can now login (account is active)
       │
       ▼
  [POST /api/v1/auth/login-web]
  Response: 403 Forbidden — "Active subscription required"
       │
       ▼
  Show "Subscription Required" page with CTA "View Plans"
       │
       ▼
  [GET /api/v1/plans] — Show pricing
       │
       ▼
  User selects plan + period (monthly/yearly)
       │
       ▼
  [POST /api/v1/subscriptions/initiate-payment]
  Body: { planId, period }
  Response: { paymentId, paymobRedirectUrl, paymobPaymentKey, planId, ... }
       │
       ▼
  Redirect browser to paymobRedirectUrl
       │
       ▼
  User pays on Paymob checkout page
       │
       ▼
  Paymob redirects back to /payments/result?success=true|false
       │
       ▼
  [POST /api/v1/auth/refresh-token] — Get new JWT with HasActiveSubscription: true
       │
       ▼
  Full dashboard unlocked!

  FLOW B: EXISTING CLINICOWNER — NO ACTIVE SUBSCRIPTION

4. JWT & CLIENT-SIDE GATING
================================================================================

  Every JWT access token now contains:

    {
      "HasActiveSubscription": "True",
      "ClinicId": "guid-here",
      "sub": "user-id",
      "role": "ClinicOwner"
    }

  DECODING THE JWT
  ─────────────────

  Web (JavaScript):
    import { jwtDecode } from 'jwt-decode';
    const token = localStorage.getItem('accessToken');
    const decoded = jwtDecode(token);
    const hasSubscription = decoded.HasActiveSubscription === 'True';

  Mobile (React Native):
    import jwtDecode from 'jwt-decode';
    const decoded = jwtDecode(accessToken);
    const hasSubscription = decoded.HasActiveSubscription === 'True';

  UI DECISIONS
  ────────────

  HasActiveSubscription   ClinicOwner Outcome           Staff/Doctor Outcome
  ─────────────────────   ──────────────────────────     ─────────────────────────
  True                    Full web dashboard             Full web dashboard
  False                   Limited view + Subscribe CTA   403 on login-web

6. PAYMENT FLOW (PAYMOB)
================================================================================

  STEP 1: INITIATE PAYMENT
  ────────────────────────

  POST /api/v1/subscriptions/initiate-payment
  Authorization: Bearer <jwt-token>
  Content-Type: application/json

  Request:
  {
    "planId": "guid",
    "period": 0    // 0 = Monthly, 1 = Yearly
  }

  Response (200 OK):
  {
    "paymentId": "guid",
    "paymobRedirectUrl": "https://accept.paymob.com/unifiedcheckout/?paymentKey=...",
    "paymobPaymentKey": "...",
    "planId": "guid",
    "planName": "Basic",
    "period": 0,
    "amount": 500,
    "currency": "EGP"
  }

  STEP 2: REDIRECT TO PAYMOB
  ──────────────────────────

  window.location.href = response.paymobRedirectUrl;

  STEP 3: PAYMENT RESULT PAGE (AFTER PAYMOB REDIRECT)
  ──────────────────────────────────────────────────

  Paymob redirects to: /api/v1/payments/result?success=true|false

  Frontend should detect this redirect:

    const urlParams = new URLSearchParams(window.location.search);
    const success = urlParams.get('success');

    if (success === 'true') {
      showSuccessMessage('Subscription activated successfully!');
      await refreshToken();     // Get new JWT with HasActiveSubscription: true
      window.location.href = '/dashboard';
    } else {
      showErrorMessage('Payment failed. Please try again.');
      window.location.href = '/plans';
    }

  STEP 4: REFRESH JWT AFTER SUCCESSFUL PAYMENT
  ────────────────────────────────────────────

  async function refreshToken() {
    const refreshToken = localStorage.getItem('refreshToken');
    const res = await fetch('/api/v1/auth/refresh-token', {
      method: 'POST',
      body: JSON.stringify({ refreshToken })
    });
    const data = await res.json();
    localStorage.setItem('accessToken', data.accessToken);
    localStorage.setItem('refreshToken', data.refreshToken);
  }

  function ProtectedRoute({ children }) {
    const token = localStorage.getItem('accessToken');
    const decoded = jwtDecode(token);

    if (decoded.HasActiveSubscription !== 'True') {
      return <Navigate to="/subscription-required" />;
    }

    return children;
  }

8. SUPERADMIN DASHBOARD PAGES
================================================================================

  8a. PENDING CLINICS PAGE
  ────────────────────────

  GET /api/v1/admin/dashboard/clinics/pending

  Display table with:
    - Clinic Name
    - Address / Phone
    - Specialization
    - Owner Name, Email (from clinicAdminId -> User)
    - Registration Date
    - Actions: [Approve] [Reject]

  Approve:  POST /api/v1/admin/dashboard/clinics/{id}/approve
  Reject:   POST /api/v1/admin/dashboard/clinics/{id}/reject
            Body: { "reason": "Missing required documents" }

  8b. PLAN MANAGEMENT PAGE (CRUD)
  ──────────────────────────────

  GET    /api/v1/admin/plans        — Fetch all plans
  POST   /api/v1/admin/plans        — Create plan (see CreatePlanCommand fields)
  PUT    /api/v1/admin/plans/{id}   — Update plan (see UpdatePlanCommand fields)
  DELETE /api/v1/admin/plans/{id}   — Delete (soft-delete) plan

  Create/Update Plan Fields:
    name, nameAr, description, descriptionAr,
    priceMonthly, priceYearly,
    maxDoctors (null = unlimited),
    maxStaff   (null = unlimited),
    features   (JSON string array),
    isActive   (boolean),
    sortOrder  (integer)

  8c. SUBSCRIPTION MANAGEMENT PAGE
  ───────────────────────────────

  GET /api/v1/admin/dashboard/subscriptions?status=&planId=&clinicId=&pageNumber=1&pageSize=20

  Filters:
    status    — SubscriptionStatus enum (0=Active, 1=Cancelled, 2=Expired, 3=Revoked)
    planId    — Filter by plan GUID
    clinicId  — Filter by clinic GUID
    pageNumber / pageSize — Pagination

  Display table:
    - Clinic Name
    - Plan Name
    - Period (Monthly/Yearly)
    - Amount
    - Start Date / End Date
    - Status
    - [Revoke] button for active subscriptions

  Revoke subscription:
    POST /api/v1/admin/dashboard/subscriptions/{id}/revoke
    (triggers Paymob refund if payment has transaction ID)

9. ERROR HANDLING
================================================================================

  HTTP STATUS CODES
  ─────────────────

  Status   Meaning                Frontend Handling
  ───────  ───────────────────    ──────────────────────────────────────────
  200      Success                Normal processing
  201      Created (register)     Show success message
  202      Accepted (register)    Show "pending approval" screen
  400      Bad Request            Show validation errors from response body
  403      Forbidden (login-web)  Redirect to "Subscription Required" page
  403      Forbidden (endpoint)   Show "Feature requires subscription" toast
  404      Not Found              Show "Not found" message
  500      Server Error           Show generic error + retry

  API RESPONSE FORMAT
  ───────────────────

  All responses follow this structure:

  {
    "success": true/false,
    "message": "Human-readable message",
    "errors": { "fieldName": ["error1", "error2"] },
    "data": { ... },
    "statusCode": 200
  }

  The API controllers use helper methods (Ok(), Created(), etc.) from
  BaseApiController which wrap responses in ApiResponse<T>.

  COMMON ERROR SCENARIOS
  ──────────────────────

  a. Login-web returns 403:
       Show "Subscription Required" page with "View Plans" button.
       Do NOT redirect to dashboard.

  b. Initiate payment returns 400:
       Possible reasons:
         - Plan not found or inactive
         - Clinic not active (pending approval)
         - Already have an active subscription
       Show the message from response.

  c. Endpoint returns 403 (RequireSubscription filter):
       The action filter checks for active subscription in DB.
       Show toast: "An active subscription is required."

10. LOCALIZATION
================================================================================

  The API uses Accept-Language header for localization.

  Default: Arabic (ar)
  English: set header Accept-Language: en

  Example:
    fetch('/api/v1/plans', {
      headers: { 'Accept-Language': 'en' }
    });

  All user-facing messages come pre-translated in the API response.
  Plan names and descriptions have separate English/Arabic fields (name, nameAr).
