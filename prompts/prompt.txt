The login-web handler already distinguishes between these cases — it returns 403 with different messages:
Scenario
User not active / pending verification
No active subscription
The frontend should check the response body to decide which screen to show:
const res = await fetch('/api/v1/auth/login-web', { method: 'POST', body: JSON.stringify(data) });

if (res.ok) {
  // Normal dashboard
} else if (res.status === 403) {
  const body = await res.json();
  if (body.message?.includes('pending') || body.message?.includes('Pending')) {
    showPendingApprovalScreen();  // Registration not yet approved by admin
  } else {
    showSubscriptionRequiredScreen();  // Already approved, just needs to subscribe
  }
}
That way, a clinic owner who registered via mobile (and is already approved) will skip the "Pending Approval" screen and go directly to subscribing, while a newly registered user will see the pending message.
